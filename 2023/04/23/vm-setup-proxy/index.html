1. 场景

一些日常开发、调试工作一般在虚拟机中进行，如需从 GitHub 下载源码，会由于网络问题导致经常下载失败，一般可以通过配置 ssh 或直接下载 zip 代码包来解决。但对于一些较大型的项目，会包含一些子模块并需要安装很多其他依赖项，如继续使用上述方式将导致将导致整个过程非常繁琐且不一定能够解决，可以通过终端配置代理的方式来解决，具体方法如下文。
